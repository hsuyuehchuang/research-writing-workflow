# Style Guide

## Main Note Defaults

- Prefer one main Markdown document per topic.
- Keep the default main body short enough for a 30-minute report.
- Use appendices for optional deep derivations.
- Explain both mathematical meaning and physical meaning when relevant.

## Writing Priorities

1. clarity
2. engineering meaning
3. mathematical correctness
4. report-ready structure
5. depth on demand

## Preferred Structure

```text
# Title
## Problem Definition
## Summary
## Background / Motivation
## Core Principle
## Mathematical Explanation
## Physical Interpretation
## Trade-offs
## Conclusion
## Appendix (optional)
```

## Appendix Rule

The appendix is not automatic.
Only add it when:

- the user asks to expand
- the omitted details would otherwise block correctness

## Publishing Rule

Markdown remains the source of truth even after LaTeX and PDF are generated.
